# ネタトレンドデイリー ワークフロー（完全自動化版）

**このワークフローを実行すると、AIが自動的に:**
1. Web検索で最新記事を30本収集
2. りゅう専用フィルターで10本に厳選
3. 読まない方がいい5本も辛口選定
4. Obsidian用マークダウンを自動生成・保存

**あなたがやること:** 確認して、自分の判断と次のアクションを追記するだけ！

---

### 1. 多角化検索（Deep Search）

以下の3方向から同時に検索をかけ、情報をクロスリファレンスする。
- **ソースA（一次情報）**: arXiv, Nature, Science, 各国政府、テック企業公式ブログ。
- **ソースB（実学）**: 実装事例、GitHubトレンド、製品リリースノート。
- **ソースC（文脈）**: 信頼できる専門家、トップティアのテック系ニュースレター。

// turbo
```bash
# 各ソースから30本程度の候補を自動収集
```

### 2. 厳格なフィルタリング（りゅう専用フィルター）

収集した候補から、以下の「編集者視点」で10本に絞り込む。
- **[Monetize]**: 企画、コンサル、執筆など、仕事のネタとして換金性が高い。
- **[Interest]**: 知的好奇心を刺激し、思考のOSをアップデートする。
- **[Trash Check]**: 釣り記事、中身のないコピペ、SEO目的のまとめサイトは強制排除。

### 3. リンク検証と「知の接続」

選定した10本について、以下のツールで検証と文脈付与を行う。

- **実働検証**: `read_url_content` 等で404や内容確認不可（有料壁等）を排除。
- **プロジェクトへの接続**: 要約するのではなく、**「この記事が、りゅうのどのプロジェクトや興味に接続されるか」**（Context）を記述する。

### 4. Obsidian用マークダウンを生成・保存（AI実行）

**出力フォーマット（Intelligence版）:**

```markdown
---
type: neta-trend-daily
date: YYYY-MM-DD
tags: [scrap, daily]
---

# ネタトレンドデイリー YYYY-MM-DD

## 🧠 今日のインテリジェンス（AIによる概況分析）
> [特定のテーマの加速や、大手メディアを外した編集方針など、AIによる分析レポートを記述]

## 🏆 選定ベスト10（検証済み）

| Rank | 💡 Hook（本質を突く煽り）                      | 💰 価値分類 | 📈 期待リターン | Action   | Source | Link   |
| ---: | :-------------------------------------------- | :--------- | :------------- | :------- | :----- | :----- |
|    1 | **[タイトル]** <br>背景とプロジェクトへの接続 | [Monetize] | ...            | **Read** | ...    | [Link] |

## 🚫 読まなくていい5本（辛口排除）
*これらに目を通す時間は「損失」です。*
1) **[タイトル]**:（理由を辛口に記述）
2) ...

## 🛠 自分の判断（3行）
- **今日読むのは：**
- **今日捨てるのは：**
- **今日のアウトプット種：**

## 🚀 次の1アクション
- [ ] 
```

### 5. ユーザーが確認・追記

生成されたファイルを開いて:
1. ベスト10を確認
2. 「自分の判断」セクションを埋める
3. 「次の1アクション」を1つだけ決める
4. 実行！

## カスタマイズのヒント

### 収集ソースを固定化する

`life/prompts/neta-trend-daily.md` の「情報源の方針」セクションに追加:

**必ず見るべきソース（例）:**
- Nature, Science, arXiv（科学）
- OpenAI Blog, Anthropic Blog, Google AI Blog（AI）
- 特定のニュースレター（例: The Batch, Import AI）

**除外すべきソース（例）:**
- まとめサイト、信頼性の低いメディア

### 重点テーマの調整

上記「重点テーマ」の優先度や内容を変更可能。
